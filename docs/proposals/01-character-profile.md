## 16종 캐릭터 로스터 확장 및 프로필 반영
### 문제 (Why)
현재 진단 퀴즈로 4종 캐릭터만 배정되어 MZ 타겟의 세분화된 성향(MBTI 스타일)을 만족시키지 못함. 16종으로 다각화하여 유저의 애착도를 높이고 소셜 공유(자랑)를 유도해야 함.

### MVP 범위 (3줄 이내)
온보딩 성격 진단 후 도출되는 캐릭터 타입을 기존 4종에서 16종으로 확장.
진단 완료 시 제공되는 카드형 결과 화면을 프로필 탭의 메인 위젯으로 동일하게 노출.
전체 16종 컬렉션 도감 화면 추가 및 미확인 타입 블라인드 처리.

### DB 스키마 변경
- `starter_character_catalog` 테이블: 기존 4종에서 16종 데이터로 로우 추가 (코드: INTJ_explorer 등).
- `profiles.explorer_type`: JSONB 밸류 내 `type_code`를 16종 식별자로 규격화.
- 새로운 컬럼이나 테이블 추가 불필요 (기존 컬럼 포맷 확장 활용).

### RPC / Edge Function 변경
- 기존 온보딩 진단 결과 매핑 로직 수정: 4분면 알고리즘을 16개 캐릭터 분기로 확장 연산.
- `get_public_profile` RPC: 공개 프로필 요청 시 유저의 `explorer_type` 상세(16종 식별자 및 일러스트 메타)를 함께 반환하도록 쿼리 조인 최적화.

### 클라이언트 변경 (화면, 컴포넌트)
- `app/(tabs)/profile.tsx`: 상단 영역에 `ProfileTypeCard` 위젯 삽입 (온보딩의 결과 카드 재사용).
- `app/(auth)/personality-quiz.tsx`: 16종 결과 진단 로직 매핑 반영.
- `src/components/character/CharacterCard.tsx`: 16종 일러스트 및 뱃지 에셋 동적 렌더링 지원.

### 공수 추정 (S/M/L/XL)
M (디자인 에셋 추가가 메인이며, 기능 구현은 기존 폼 활용으로 비교적 단기)

### 리스크 (Top 2)
- 16종 에셋(일러스트, 스프라이트) 추가로 인한 초기 앱 번들 사이즈 급증.
- 기존 가입 유저의 4종 타입 데이터를 16종으로 자동 매핑(마이그레이션) 시 정합성 문제.

### 명시적으로 MVP에서 제외하는 것
- 이미 선택항목이 고정된 유저의 자유로운 성향 재검사 기능 (가입 시 1회 고정 원칙 유지).
- 다른 유저의 16종 캐릭터와 상호작용하는 다이내믹 UI 애니메이션.
