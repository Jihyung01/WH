# WhereHere — 스크린샷 가이드

앱스토어 제출에 필요한 스크린샷 촬영 가이드입니다.

---

## 스크린샷 사양

### iOS App Store
| 기기 | 해상도 | 필수 여부 |
|---|---|---|
| iPhone 6.9" (15 Pro Max 등) | 1320 × 2868 px | **필수** |
| iPhone 6.7" (14 Plus 등) | 1290 × 2796 px | 권장 |

|---|---|---|

## 권장 스크린샷 순서 (총 6장)

> 첫 번째 스크린샷이 가장 중요합니다. 스토어 목록에서 가장 먼저 보입니다.

### 1번 — 지도/탐험 화면 ⭐ (메인)
**캡처 화면**: 홈 지도 탭
**표시 내용**: 서울 지도 위에 주변 이벤트 핀들이 보이는 장면
**추가 텍스트 오버레이 (선택)**:
```
여기, 지금 — 탐험을 시작하세요
서울 곳곳의 숨겨진 명소를 발견하세요
```

---

### 2번 — 이벤트 상세 / 미션 화면
**캡처 화면**: 이벤트 상세 또는 미션 목록
**표시 내용**: 미션 카드들이 나열된 화면, 완료/진행중 상태 포함
**추가 텍스트 오버레이 (선택)**:
```
다양한 미션에 도전하세요
포토, 퀴즈, 체크인 미션
```

---

### 3번 — 캐릭터 화면
**캡처 화면**: 캐릭터 프로필 또는 캐릭터 선택 화면
**표시 내용**: 4가지 캐릭터 중 하나가 스탯과 함께 표시
**추가 텍스트 오버레이 (선택)**:
```
나만의 동반자를 키우세요
탐험력, 매력, 체력, 행운
```

---

### 4번 — 체크인 / 인증 화면
**캡처 화면**: 체크인 성공 또는 미션 완료 모달
**표시 내용**: 성공 애니메이션, 획득한 XP/코인/배지
**추가 텍스트 오버레이 (선택)**:
```
실제 방문으로 보상 획득
XP, 코인, 한정 배지를 모으세요
```

---

### 5번 — 뱃지 / 보상 화면
**캡처 화면**: 뱃지 컬렉션 또는 프로필 성취 화면
**표시 내용**: 다양한 배지들이 그리드로 나열
**추가 텍스트 오버레이 (선택)**:
```
수집의 재미
지역별, 미션별 한정 배지를 모두 모아보세요
```

---

### 6번 — 온보딩 / 캐릭터 생성 화면
**캡처 화면**: 캐릭터 생성 3단계 중 한 장면
**표시 내용**: 캐릭터 선택 또는 이름 입력 단계
**추가 텍스트 오버레이 (선택)**:
```
지금 바로 시작하세요
카카오 로그인 후 3분이면 탐험 준비 완료
```

## Google Play 피처드 이미지 (1024 × 500 px)

스크린샷이 아닌 **배너 이미지**입니다. 앱스토어 상단에 표시됩니다.

**권장 구성**:
- 배경: 앱 테마색 (#0F172A 다크 또는 #1AAD8A 그린)
- 왼쪽: 앱 이름 "WhereHere" + 슬로건 "여기, 지금, 탐험을"
- 오른쪽: 주요 화면 목업 1~2개 (폰 프레임 포함)

**간단 제작 도구**: Canva, Figma (1024×500 템플릿 검색)

## 스크린샷 촬영 팁

### Expo/React Native 기기 촬영
1. 실제 기기에서 `npx expo start --no-dev` 로 프로덕션 빌드 실행
2. iOS: 전원 + 볼륨업 동시 누르기
3. Android: 전원 + 볼륨다운 동시 누르기
4. 상태바 시간을 9:41로 고정하면 깔끔합니다 (iOS 설정 > 개발자 옵션)

### 텍스트 오버레이 추가 (선택)
스크린샷에 설명 텍스트를 덧붙이면 전환율이 높아집니다.
- **Canva**: 무료, 앱스토어 스크린샷 템플릿 다수 제공
- **Figma**: 무료, 정밀한 제어 가능
- **AppLaunchpad** / **Shotbot**: 스크린샷 자동 프레이밍 도구

### 권장 폰 목업 프레임
- iPhone 15 Pro (블랙 또는 티타늄)
- 무료 Figma 커뮤니티: "iPhone 15 Pro mockup" 검색

## 앱 아이콘 체크리스트

기존 `assets/icon.png` 파일 확인 사항:

| 항목 | 요구사항 | 확인 |
|---|---|---|
| iOS 아이콘 크기 | 1024 × 1024 px | `assets/icon.png` |
| iOS 아이콘 형식 | PNG, 투명 배경 없음, 둥근 모서리 없음 (OS가 자동 처리) | - |
| Android 아이콘 (Adaptive) | foreground + background 분리 | `assets/android-icon-*.png` |
| Android 아이콘 배경색 | `#1AAD8A` (app.json에 설정됨) | ✓ |
| Google Play 아이콘 | 512 × 512 px PNG | icon.png를 리사이즈 |

> `icon.png`가 1024×1024이면 EAS Build가 모든 크기를 자동 생성합니다.

## 제출 전 최종 체크리스트

### 공통
- [ ] 개인정보처리방침 URL 접근 가능 확인 (`https://wherehere.app/privacy`)
- [ ] 이용약관 URL 접근 가능 확인 (`https://wherehere.app/terms`)
- [ ] 고객센터 이메일 확인 (`support@wherehere.app`)
- [ ] 스크린샷 최소 2장 준비 (권장 6장)
- [ ] 앱 설명 복사 완료

### iOS 전용
- [ ] Apple Developer 계정 활성화 ($99/년)
- [ ] App Store Connect에서 앱 생성 (`com.wherehere.app`)
- [ ] 번들 ID가 Xcode/EAS 설정과 일치 확인
- [ ] TestFlight 내부 테스트 완료
- [ ] 연령 등급 설문 완료 (예상: 4+)
- [ ] 암호화 사용 여부 답변 (ITSAppUsesNonExemptEncryption: false ✓)

### Android 전용
- [ ] Google Play Console 계정 생성 ($25 1회)
- [ ] 앱 서명 키 생성 및 보관 (분실 시 업데이트 불가)
- [ ] 피처드 이미지 (1024 × 500 px) 준비
- [ ] 연령 등급 설문 완료 (예상: 전체 이용가)
- [ ] 데이터 보안 섹션 입력 (위치, 개인정보 수집 항목 고지)
